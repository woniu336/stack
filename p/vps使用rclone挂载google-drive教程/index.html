2023-08-09

其实 rclone 挂载 Google Drive 网上的教程已经很多了，但是大多数的教程都有一点点过时，与现在的实际操作有一点出入，故整理了当前的挂载操作，以防忘记。

一、 安装并配置 rclone

首先在 vps 上一键安装 rclone：

1

curl https://rclone.org/install.sh | sudo bash

接下来在本地电脑上下载命令行操作的 rclone：

访问 rclone 下载地址，选择您的操作系统下载相应的 zip 包并解压。一会需要用得着。

软件准备好中，在 vps 上开始配置，执行：

1

rclone config

二、下载rclone windows版

https://rclone.org/downloads/

下载在本地的 rclone 压缩包解压后，你应该能看到一个 rclone.exe，类似下图

空白处右键–在此处打开命令行窗口，或者cmd进入你的rclone目录

打开cmd,粘贴前面复制的代码回车（红色框起来的地方）

（浏览器自动打开进入到谷歌账号登录界面，并选择同意授权。由于是新申请的 API，所以谷歌会提醒该应用未经验证。但是左下角有一行小灰字，点击选择继续就能授权）

授权成功浏览器会提示 success：

此时返回命令行，等待几秒就能看见授权的 code 了。

如果出现下面这种情况（waiting for code），请往下看第五点

复制授权，回到vps终端，粘贴到 rclone 中。红色框起来的地方

接下来 rclone 会询问是否为团队盘：选择n

y) Yes
n) No (default)

如果您要挂载的就是团队盘，那么选择 y，不是的话就选择 n

此时配置就已经结束了，退出 clone，开始挂载。

三、 挂载 Google Drive

首先新建一个文件夹用于挂载：

1

mkdir /google

开始挂载：

rclone mount 283pan: /google --allow-other --allow-non-empty --vfs-cache-mode writes &

其中 283pan 是 rclone 配置时输入的配置名称，/google 是挂载目录

​ 挂载出错请往下看第四点

检查挂载：

1

df -h

卸载挂载：（/google 为挂载目录）

1

fusermount -u /google

 

四，挂载出错解决

可能会报错：

1

Fatal error: mount not ready

一般是因为缺少依赖导致的，我们选择安装

centos 系使用：

1

yum install -y fuse fuse3 或者 yum install fuse3

debian 系使用：

1

apt-get install fuse3

再次执行挂载命令，如果没有报错，就是挂载成功了。

五，解决Rclone生成不了token code的问题

 

这是因为开了科学上网，但是cmd没走代理。我用的是小狐狸，端口是7890，如果和我一样，那就复制了往里敲

1
2

set http_proxy=socks5://127.0.0.1:7890
set https_proxy=socks5://127.0.0.1:7890

配置完之后，我们再跑一次 rclone authorize，code显示出来了

取消sock5代理

1
2

set http_proxy=
set https_proxy=

其实 `rclone` 挂载 `Google Drive` 网上的教程已经很多了，但是大多数的教程都有一点点过时，与现在的实际操作有一点出入，故整理了当前的挂载操作，以防忘记。

## 一、 安装并配置 rclone

`首先在 vps 上一键安装 rclone：`

```bash
curl https://rclone.org/install.sh | sudo bash
```

接下来在本地电脑上下载命令行操作的 rclone：

访问 [rclone 下载地址](https://rclone.org/downloads/)，选择您的操作系统下载相应的 zip 包并解压。一会需要用得着。

软件准备好中，`在 vps 上开始配置`，执行：

```bash
rclone config
```

`选择 n`，新建一个

```html
No remotes found, make a new one?
n) New remote
s) Set configuration password
q) Quit config
n/s/q> n
```



输入新建的名称，随意写:比如 `283pan`

```html
Enter name for new remote.
name>283pan
```

选择要挂载的网盘类型，`选择18`，`Google Drive`



填写OAuth client ID，输入`client_id`、`client_secret`



选择Rclone对Google Drive 谷歌云盘的操作权限，我这里选择1，完整的访问权限




回车默认


## 三、 挂载 Google Drive

首先新建一个文件夹用于挂载：

```
mkdir /google
```

开始挂载：

`rclone mount 283pan: /google --allow-other --allow-non-empty --vfs-cache-mode writes &`

其中 `283pan` 是 rclone 配置时输入的配置名称，`/google` 是挂载目录

---

​                             挂载出错请往下看第四点

---



检查挂载：

```bash
df -h
```

![image-20230810011416520](https://r2.leshans.eu.org/2023/08/626fa1df5fe52993cd3bd8b4868d373a.webp)

卸载挂载：（/google 为挂载目录）

```bash
fusermount -u /google
```

## 四，挂载出错解决

可能会报错：

```bash
Fatal error: mount not ready
```

一般是因为缺少依赖导致的，我们选择安装

centos 系使用：

```bash
yum install -y fuse fuse3 或者 yum install fuse3
```

debian 系使用：

```bash
apt-get install fuse3
```

再次执行挂载命令，如果没有报错，就是挂载成功了。





## 五，解决Rclone生成不了token code的问题

![image-20230810012334426](https://r2.leshans.eu.org/2023/08/20c21f5c2bd1496164aecb572c258dd2.webp)



&nbsp;

这是因为开了`科学上网`，但是cmd没走代理。我用的是小狐狸，端口是`7890`，如果和我一样，那就复制了往里敲



```bash
set http_proxy=socks5://127.0.0.1:7890
set https_proxy=socks5://127.0.0.1:7890
```



配置完之后，我们再跑一次 `rclone authorize`，code显示出来了

取消sock5代理

```bash
set http_proxy=
set https_proxy=
```
